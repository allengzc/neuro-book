我发现一个现象：类似 director，simulator 这些资产管理任务 agent，甚至 writer 也是。他们的协作模式很相似：

例如上级（指的是用户或者调用方agent）派发任务：设计下一章的剧情。他不能直接自主设计好剧情，然后直接落实到文件。因为这样会导致他的任务和上级的需求偏离，他应该询问（或者拷问）上级，对其需求，持续讨论。最后等上级觉得可以了然后最后再总结落库

我觉得这里存在两种不同的场景：

1. 交流模式，或者半自动模式（小说模式，Agent 必须反复追问和用户需求对其）
2. 全自动模式（更加适合无用户参与的情况，例如 Roleplay 模式 Agent 自主设计剧情）
